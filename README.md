# Лабораторная работа №3: Сравнение производительности коллекций

## Описание проекта

Данный проект предназначен для практического сравнения производительности двух популярных реализаций интерфейса `List` в Java: `ArrayList` и `LinkedList`.

Программа выполняет замеры времени для 9 основных сценариев: операций добавления, чтения и удаления в начале, в середине и в конце списка.

## Технологии

*   Java 24 (Corretto)
*   Maven

## Как запустить проект

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/Syxarik2005/java-lab3.git
    cd java-lab3
    ```
2.  **Запуск:**
    *   **Способ 1 (Рекомендуемый):** Открыть проект в любой IDE, поддерживающей Maven (например, IntelliJ IDEA), и запустить метод `main` в классе `Main.java`.
    *   **Способ 2 (Командная строка):** Выполнить команду Maven, которая скомпилирует и запустит программу.
        ```bash
        mvn compile exec:java -Dexec.mainClass="com.sergey.Main"
        ```

## Результаты тестов и их анализ

Ниже представлены реальные результаты выполнения тестов и их теоретическое обоснование. Значение "0 мс" означает, что операция выполнялась настолько быстро, что ее время было меньше одной миллисекунды.

### Тестирование ArrayList

| Метод                     | Количество        | Время (ms) |
| ------------------------- | ----------------- | ---------- |
| Добавление в конец        | 100000            | 2          |
| Добавление в середину     | 1000              | 7          |
| Добавление в начало       | 1000              | 7          |
| Чтение с начала           | 100000            | 0          |
| Чтение из середины        | 100000            | 0          |
| Чтение с конца            | 100000            | 0          |
| Удаление с конца          | 1000              | 0          |
| Удаление из середины      | 1000              | 3          |
| Удаление с начала         | 1000              | 6          |

**Анализ:**
*   **`get` (чтение):** `ArrayList` показывает феноменальную скорость. Время доступа к любому элементу (в начале, середине или конце) составило **0 мс**, что идеально демонстрирует его сложность `O(1)` (константное время).
*   **Операции в конце:** Добавление и удаление с конца также очень быстрые.
*   **Операции в начале/середине:** Это худший сценарий (`O(n)`). Даже для 1,000 операций мы видим измеримое время (`3-7 мс`), так как `ArrayList` вынужден сдвигать сотни тысяч элементов при каждой такой операции.

### Тестирование LinkedList

| Метод                     | Количество        | Время (ms) |
| ------------------------- | ----------------- | ---------- |
| Добавление в конец        | 100000            | 2          |
| Добавление в середину     | 1000              | 71         |
| Добавление в начало       | 1000              | 0          |
| Чтение с начала           | 100000            | 3          |
| Чтение из середины        | 100000            | 5795       |
| Чтение с конца            | 100000            | 2          |
| Удаление с конца          | 1000              | 0          |
| Удаление из середины      | 1000              | 58         |
| Удаление с начала         | 1000              | 0          |

**Анализ:**
*   **Операции в начале/конце:** `LinkedList` здесь абсолютный чемпион. Добавление и удаление занимают **0 мс**, так как требуют только переназначения пары ссылок (`O(1)`).
*   **`get` (чтение из середины):** Это его ахиллесова пята (`O(n)`). Поиск элемента в середине занял **почти 6 секунд**, так как `LinkedList` последовательно прошел по 50,000 ссылкам.
*   **Операции в середине:** Также очень медленные (`71 мс` и `58 мс` для 1,000 операций). Основное время тратится не на саму вставку/удаление, а на **поиск** нужного элемента в середине списка.

## Вывод

Проведенные тесты наглядно демонстрируют фундаментальные различия в устройстве `ArrayList` и `LinkedList`.

*   **`ArrayList`** — идеальный выбор, когда основной операцией является частое чтение данных по индексу.
*   **`LinkedList`** — незаменим, когда требуется множество операций вставки или удаления в начало/конец списка (например, при реализации очередей или стеков).
